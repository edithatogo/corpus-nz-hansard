# Spec: W3C Time Temporal Model

## Goal

Model parliamentary periods, sittings, offices, party memberships, and role intervals using W3C Time-compatible structures.

## MoSCoW Requirements

### Must

- Define interval/instant representation for parliamentary temporal data.
- Align with party attribution, member identity, sittings, and RDF linked-data needs.
- Validate date precision, open intervals, overlaps, and source provenance.

### Should

- Emit RDF-ready mappings and JSON schema guidance.
- Include tests for party switches, office tenures, and sitting dates.

### Could

- Add timeline examples in the documentation portal.

### Won't

- Force incomplete historical dates into false precision.

## Acceptance Criteria

- Temporal model docs, schemas/mappings, and validation fixtures exist.
