# Spec: Upstream Contribution Packages

## MoSCoW Requirements

### Must

- Identify the current contribution path for each target before preparing a submission.
- Package validation evidence, samples, mapping notes, and known exclusions.
- Keep local publication separate from external acceptance.
- Track maintainers' feedback and decisions as evidence.

### Should

- Prepare ParlaMint-NZ samples first because ParlaMint is the core scholarly endpoint target.
- Prepare CAP/ParlaCAP mapping notes after topic contracts exist.
- Prepare mySociety/PublicWhip-style references only if maintainers accept the scope.

### Could

- Add issue or pull-request templates for external submissions.

### Won't

- Open upstream PRs with unvalidated or exploratory artifacts.
- Claim external project acceptance until merged or otherwise confirmed.

## Acceptance Criteria

- Contribution-readiness manifest exists for each target endpoint.
- External submissions, if made, have links and evidence recorded.

## Dependencies

- Depends on endpoint validation, historical coverage audit, release ladder, known-exclusions documentation, and maintainer-specific contribution mechanism checks.
